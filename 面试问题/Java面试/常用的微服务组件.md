目前主流的方案主要分为 **Spring Cloud (Netflix/原生)** 和 **Spring Cloud Alibaba** 两大阵营。以下是常用的核心组件分类：

---

### 1. 服务注册与发现 (Service Registry)

**作用**：就像一个“通讯录”，记录每个服务实例的 IP 和端口。

- **Nacos (推荐)**：阿里巴巴开源，集成了注册中心和配置中心，支持 CP 和 AP 模式，国内应用极广。
    
- **Eureka**：Netflix 出品，虽然进入维护期，但很多老项目仍在沿用。
    
- **Consul / Zookeeper**：提供强一致性的注册方案。
    

### 2. API 网关 (API Gateway)

**作用**：系统的入口，负责路由转发、权限校验、限流、日志监控。

- **Spring Cloud Gateway (推荐)**：基于响应式编程，性能高，是目前 Spring 生态的首选。
    
- **Zuul**：Netflix 的老牌网关，Zuul 1.x 性能较差，已逐渐被淘汰。
    
- **APISIX / Kong**：基于 Nginx/OpenResty 的高性能网关，适合云原生场景。
    

### 3. 配置中心 (Configuration Management)

**作用**：集中管理所有微服务的配置，支持不重启应用动态刷新配置。

- **Nacos (推荐)**：配置与注册一体化，非常方便。
    
- **Apollo (阿波罗)**：携程开源，功能极其强大，支持治理、权限审核等，适合大型企业。
    
- **Spring Cloud Config**：配合 Git 使用，但相对笨重。
    

### 4. 服务调用与负载均衡 (RPC & Load Balance)

**作用**：解决服务之间如何像调用本地方法一样互相通信，并平均分配请求压力。

- **OpenFeign**：声明式 HTTP 客户端，代码简洁。
    
- **Dubbo**：高性能 RPC 框架，支持多种协议。
    
- **LoadBalancer**：Spring Cloud 原生负载均衡器（替代了之前的 Ribbon）。
    

### 5. 熔断、降级与限流 (Resilience)

**作用**：防止某个服务故障导致整个系统“雪崩”。

- **Sentinel (推荐)**：阿里巴巴出品，带控制台，支持流量控制、熔断降级、热点参数限流。
    
- **Hystrix**：老牌组件，已停更，但其“隔离”和“断路器”思想是面试必考。
    
- **Resilience4j**：轻量级熔断器，Spring Cloud 官方推荐的 Hystrix 替代品。
    

### 6. 链路追踪与监控 (Observability)

**作用**：排查一个请求跨越多个服务时，哪一环慢了或出错了。

- **SkyWalking (推荐)**：国产之光，无侵入（通过字节码增强），报表非常漂亮。
    
- **Zipkin / Sleuth**：经典的链路追踪组合。
    
- **Prometheus + Grafana**：用于监控服务器指标（CPU、内存、QPS）。
    

---

### 核心组件对比对照表

|**功能模块**|**Spring Cloud (Netflix/原生)**|**Spring Cloud Alibaba**|
|---|---|---|
|**服务注册**|Eureka / Consul|**Nacos**|
|**配置中心**|Config / Apollo|**Nacos**|
|**服务调用**|OpenFeign|Dubbo / OpenFeign|
|**负载均衡**|LoadBalancer|LoadBalancer|
|**熔断限流**|Resilience4j / Hystrix|**Sentinel**|
|**分布式事务**|(无成熟原生方案)|**Seata**|
